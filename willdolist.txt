        get_daily_summary, detect_business_anomalies, analyze_sku_trends,

      - 同样需要支持 tool_choice="auto"


Phase 1：核心 ReAct 循环（MVP）
--------------------------------
[ ] 1.1 新建 backend/app/agents/react_agent.py
      - 写 SYSTEM_PROMPT：定义 Agent 角色、可用工具列表、决策规则
      - 实现 react_loop_stream() 函数：
        a. 初始化 messages = [system_prompt, context, user_query]
        b. 进入循环（最多 15 轮迭代）：
           1. 调 LLM，传入 messages + tools + tool_choice="auto"
           2. 如果 LLM 返回 tool_calls → 逐个执行 → 结果塞回 messages
           3. 如果 LLM 返回纯文本 → 认为已完成，输出最终回答
           4. 每步通过 SSE yield 推送给前端
        c. 设置 循环终止条件：
          - LLM 不再调用工具（输出纯文本）
          - 达到最大迭代次数 15
          - Token/时间超限
      - 每轮记录 trace_steps，最终写入 AgentTrace

[ ] 1.2 新增 SSE 事件类型
      - tool_call:     LLM 决定调用的工具名 + 参数
      - tool_result:   工具执行结果
      - thinking:      LLM 的推理过程（如果有 reasoning 模型）
      - final_answer:  最终回答（不再调工具时输出）
      - error:         异常
      这些事件类型和前端已有的基本兼容

[ ] 1.3 新增 API 端点 /api/agent/stream-react
      - 在 backend/app/api/agent.py 中新增路由
      - 调用 react_loop_stream() 而非原来的 run_agent_stream()
      - 参数不变：query, store_id, start_date, end_date
      - 返回 EventSourceResponse

Phase 3：HITL（人在回路）融入 ReAct
-----------------------------------
